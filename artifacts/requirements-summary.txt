Interactive Healthcare Website

Pages & Navigation:
- Home page: hero section with intro and CTA buttons.
- More Information page: services overview with interactivity.
- Contact Us page: form with client-side validation and feedback.
- Sticky navbar highlights active page and includes back-to-top button.
- Back-to-top button appears after scrolling.

Contact Us Form:
- Fields: Full Name (required), Email (required, valid), Phone (optional, auto-format), Message (required, 20-1000 chars).
- Real-time validation with inline messages; disable Submit until valid.
- On submit: show loading spinner, simulate success/failure (800-1500ms).
- Success: clear form, focus first field, show dismissible success banner.
- Failure: keep inputs, focus first invalid field, show error banner.
- Accessibility: proper labels, ARIA aria-invalid, keyboard navigation, clear focus styles.

More Information Page:
- Search: filters service cards by title/description; debounced at ~250ms.
- Filters: tag chips (multi-select) combined with search.
- Sort: dropdown to sort cards by Alphabetical or Popularity.
- FAQ accordion: expand/collapse answers; one open at a time; persist state via localStorage.
- Tabbed content: switch between Services, Insurance, and Locations with smooth transitions.
- Optional map embed: highlight map marker on location card click.

Shared UI/Interactions:
- Theme toggle: light/dark mode persisted in localStorage.
- Toast notifications: reusable component; auto-dismiss with manual close.
- Smooth scrolling for in-page anchors; respect prefers-reduced-motion.
- Lazy-load images with loading='lazy'; show skeleton loaders until ready.

Accessibility & Performance:
- WCAG 2.1 AA: color contrast, focus order, skip-to-content link, ARIA roles.
- Keyboard-first: all interactive elements operable by keyboard.
- Performance: debounce search, avoid heavy libraries, minimize reflow.
- Test on mobile-first responsive layout (320px+).

Non-Functional Requirements:
- SEO: meaningful titles, meta description, favicon, structured headings.
- Content: placeholder copy/images; no real personal data stored/sent.
- Mobile-first responsive design; test on Chrome/Edge/Firefox/Safari.

Analytics:
- Log events to window.analyticsLog with { type, timestamp, payload }.
- Track events: form_submit_success, form_submit_error, search, filter_change, theme_toggle.
- Hidden debug panel: toggle via keyboard (e.g., Ctrl+Shift+D) to view events.

Acceptance Tests (Playwright):
- Navbar: sticky, highlights active page, back-to-top appears after scroll.
- Theme toggle: persists across refresh; respects reduced motion.
- Contact form: validation messages, disabled submit when invalid, success clears inputs, failure focuses first invalid field.
- More Info: search filters cards, filters/sort work together, accordion state persists, tabs switch views, skeleton loaders replaced.
- Accessibility: Tab cycles logically; ARIA attributes on accordion/tabs.

- All acceptance tests pass.
