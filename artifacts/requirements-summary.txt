Interactive Healthcare Website

Pages & Navigation:
- Home page with hero section, intro, and CTA buttons linking to other pages.
- More Information page with services overview and interactive components.
- Contact Us page with a form and interactive feedback.
- Sticky navigation bar highlights active page and includes 'Back to top' button.
- Smooth scrolling for in-page anchors; respects reduced motion.

Contact Us Form:
- Fields: Full Name (required), Email (required, valid), Phone (optional, auto-format), Message (required, 20-1000 chars).
- Real-time validation with inline messages; disable Submit until valid.
- Show loading spinner on submit; simulate success/failure after 800-1500ms.
- Success: clear form, focus first field, show dismissible banner, log analytics.
- Failure: keep inputs, focus first invalid field, show error banner.
- Accessibility: proper labels, ARIA attributes, keyboard navigation, clear focus styles.

More Information Page:
- Search input filters service cards by title/description; debounce ~250ms.
- Tag chips for multi-select filters (e.g., Primary Care, Pediatrics).
- Sort dropdown: Alphabetical or Popularity (mock data).
- FAQ accordion: expand/collapse answers; one open at a time; persist state via localStorage.
- Tabbed content: switch between Services, Insurance, and Locations with smooth transitions.
- Optional map embed: highlight map marker on location card click.

Shared UI/Interactions:
- Theme toggle: light/dark mode persisted in localStorage.
- Reusable toast notifications for success/error/info; auto-dismiss with manual close.
- Lazy-load images with skeleton loaders until content is ready.
- Accessibility: WCAG 2.1 AA compliance, ARIA roles, skip-to-content link.
- Keyboard-first experience: all elements operable by keyboard.

Performance:
- Debounce search; avoid heavy libraries; minimize reflow.
- Mobile-first responsive layout (320px to desktop).
- Test on latest Chrome, Edge, Firefox, Safari.

Analytics:
- Log events to window.analyticsLog with { type, timestamp, payload }.
- Track actions: form_submit_success, form_submit_error, search, filter_change, theme_toggle.
- Hidden debug panel toggled via keyboard (e.g., Ctrl+Shift+D) to view events.

Acceptance Tests:
- Navbar is sticky, highlights active page, and back-to-top appears after scroll.
- Theme toggle persists across refresh; respects reduced motion.
- Contact form: validation messages, disabled Submit when invalid, success clears inputs, failure preserves inputs.
- More Info: search filters cards; filters/sort work together; accordion persists state; tabs switch views.
- Accessibility: Tab cycles logically; ARIA attributes on accordion and tabs.
