Interactive Healthcare Website

Pages & Navigation:
- Home page: hero section with intro and CTA buttons linking to other pages.
- More Information page: services overview with interactive components.
- Contact Us page: form with client-side validation and feedback.
- Sticky navigation bar highlights active page and includes 'Back to top' button.
- Smooth scrolling for in-page anchors; respects reduced motion settings.

Contact Us Form:
- Fields: Full Name (required), Email (required, valid format), Phone (optional, auto-format), Message (required, 20-1000 chars).
- Real-time validation: inline messages; disable Submit until required fields are valid.
- Submission UX: loading spinner on submit; simulate success/failure (800-1500ms).
- Success: clear form, focus first field, show dismissible banner, log analytics event.
- Failure: keep inputs, focus first invalid field, show error banner with guidance.
- Accessibility: labels, ARIA aria-invalid, keyboard navigation, clear focus styles.

More Information Page:
- Search: client-side input filters service cards by title/description (debounced ~250ms).
- Filters: tag chips (multi-select) combined with search functionality.
- Sort: dropdown to sort cards by Alphabetical or Popularity (mock data).
- FAQ accordion: expand/collapse answers; one open at a time; persist state via localStorage.
- Tabbed content: switch between Services, Insurance, and Locations with smooth transitions.
- Optional map embed: static image/iframe; clicking location highlights map marker.

Shared UI/Interactions:
- Theme toggle: light/dark mode persisted in localStorage.
- Toast notifications: reusable component for success/error/info; auto-dismiss with manual close.
- Lazy-load images with loading='lazy'; use skeleton loaders until content is ready.

Accessibility & Performance:
- WCAG 2.1 AA: color contrast, focus order, skip-to-content link, ARIA roles.
- Keyboard-first: all interactive elements operable by keyboard.
- Performance: debounce search, minimize reflow, avoid heavy libraries.
- Test mobile-first responsive layout (320px to desktop) on major browsers.

Non-Functional Requirements:
- SEO basics: meaningful titles, meta description, favicon, structured headings.
- Content: placeholder copy/images; no real personal data stored or sent.

Analytics:
- Log events to window.analyticsLog with {type, timestamp, payload}.
- Track events: form_submit_success, form_submit_error, search, filter_change, theme_toggle.
- Hidden debug panel: toggle via keyboard (e.g., Ctrl+Shift+D) to view recent events.

Acceptance Tests:
- Navigation: sticky navbar, highlights active page, back-to-top appears after scroll.
- Theme toggle: persists across refresh; respects reduced motion.
- Contact form: validation messages, disabled submit when invalid, success clears inputs, failure focuses first invalid field.
- More Info: search filters cards; filters/sort work together; accordion state persists; tabs switch views; skeleton loaders replaced.
- Accessibility: Tab cycles logically; ARIA attributes on accordion and tabs.

- All acceptance tests pass.
